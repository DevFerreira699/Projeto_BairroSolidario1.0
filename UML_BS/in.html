📊 Documentação UML - Bairro Solidário

Sistema de Segurança Comunitária para o Bairro Solidário

📂 Arquitetura do Sistema

O sistema **Bairro Solidário** é estruturado em camadas. Cada camada tem uma responsabilidade específica.

Arquitetura:
    [ Apresentação ] → [ Controle ] → [ Negócio ] → [ Dados ]

Apresentação: Interface com o usuário.

Controle: Processamento das interações.

Negócio: Regras de segurança comunitária.

Dados: Armazenamento das informações.

🔸 Diagrama de Classes

O diagrama de classes descreve as principais entidades do sistema e seus atributos.

Classes principais:
        Morador
        - id
        - nome
        - endereço
        - telefone

        Ocorrência
        - id
        - tipo (roubo, invasão, barulho)
        - status (aberta, resolvida)
        - descrição
        - data
        - localização

        Alerta
        - id
        - mensagem
        - data

        Comentário
        - id
        - texto
        - data

🔸 Casos de Uso

Os casos de uso descrevem as funcionalidades do sistema e as interações dos usuários com ele.

Casos de Uso principais:
        1. Morador → Sistema: Registrar ocorrência
        2. Sistema → Banco de Dados: Salvar ocorrência
        3. Sistema → Moradores: Enviar alerta
        4. Moradores → Sistema: Visualizar ocorrência

Esses casos de uso representam o fluxo básico do sistema, envolvendo o registro de ocorrências, notificações e visualização.

🔸 Diagrama de Sequência

O diagrama de sequência mostra como os objetos interagem em uma sequência de eventos.

Exemplo de sequência de registro de ocorrência:
        Morador → Sistema: Registrar ocorrência
        Sistema → Banco de Dados: Salvar ocorrência
        Sistema → Moradores: Enviar alerta
        Moradores → Sistema: Visualizar ocorrência

Este diagrama descreve o fluxo de dados entre o **Morador**, **Sistema** e **Banco de Dados**.

📜 Regras de Negócio

As principais regras de negócio do sistema são:

  • Cadastro de Moradores: Moradores devem se cadastrar para registrar ocorrências.
  • Status das Ocorrências: As ocorrências podem ter os status **aberta**, **em análise** ou **resolvida**.
  • Envio de Alertas: Somente administradores podem enviar alertas para todos os moradores.
  • Comentários nas Ocorrências: Moradores podem comentar em ocorrências para confirmar que viram o evento.

✅ Conclusão

O **Bairro Solidário** oferece uma plataforma robusta para aumentar a segurança da comunidade. Ele permite o registro de ocorrências em tempo real, o envio de alertas para os moradores e a interação da comunidade para confirmar e comentar sobre eventos suspeitos.
